JVM 자바 운영체제 
 OS에 종속적이지 않다는 특징을 가지고 있다. 
OS에 종속받지 않고 실행되기 위해선 OS 위에서 Java 를 실행시킬 무언가가 필요하다. 
그게 바로 JVM이다.

OS에 종속적이지 않고, Java 파일 하나만 만들면 어느 디바이스든 JVM 위에서 실행할 수 있다.

자바 개발도구와 실행 환경
1. JDK 
	- 자바를 이용해 프로그램을 개발하는 데 필요한 도구를 모아 둔 집합

2. JRE 
	- 완성된 프로그램을 실행하는 데 필요한 환경(라이브러리 포함) JVM+라이브러리 파일
	- JDK를 가져와 프로그램을 오나성하고 JRE위에서 실행하면 돌아가는 것
3. JVM 
	- java 프로그램을 실행해 줌 운영체제에 의존적
	- 실제 운영체제를 대신해서 자바 프로그램을 실행하는 가장 운영체제 역할 

자바 소스코드 -> 바이트 코드(클래스 파일) -> jvm -> 실행파일(기계어) -> 실행
Hello.java     컴파일     Hello.class                                    Hello.exe
 	javac.exe(컴파일러)                                     java.exe

1. 주석
	- 설명글 : 실행하는데 지장 없음
		범위 주석 /* 주석글 */
		한줄 주석 // 주석글 또는 Ctrl + /

2. 단축키 
	글자일부 입력+ctrl+space bar 
	실행 : ctrl + f11 런타임
	크기 : ctrl + +/-  
	복사 : 드래그 한 내용 ctrl + alt + 방향키
	이동 : 드래그 한 내용 alt + 방향키 (위/아래) 
	삭제 : ctrl + d
	소스코드 정리 : 전체 선택 ctrl + shift + f (소스코드 좋게 정리)

3. 출력 메소드
	println() : 괄호내 내용 출력 및 행바꿈
	print() : 괄호내 내용 출력
	printf() : 괄호 내 내용을 포맷형식에 맞춰 출력 줄바꿈 x
		
 
6. 문장의 끝
	자바는 ; 가 있어야 문장이 끝났다고 생각한다.

7. 제어 문자
	- 반드시 " " 안에 넣는다.
	- \n : 줄바꿈
	- \t : 일정한 간격 
	- \" : "
	- \' : '
	- \\ : \

8. 문자열 연결 연산자(+)
	10 + 20 = 30
	가 + 10 = 가10

9. 개발자간의 관례 명명 규칙
	클래스명 첫글자 대문자
	클래스명 제외한 것은 첫글자 소문자
	
	캐멀케이스 : 두단어 이상이 조합되었을때 두 번째 단어부터 첫글자 대문자
		showInfoVariable() 메서드
		TestExamVariable 클래스
		
	스네이크 케이스 : 두단어 조합시 _ 묶는다.
		show_info_variable()
		
	헝가리언 표기법 : 단어 앞에 자료형 붙여서 어떤 자료형인지 알려준다,.
		intAge(정수), dSpeed(실수) 

	모든 이름은 문자로 시작하고 숫자,특수문자로 시작 안되지만 밑줄(_)은 가능하다.
	특수문자 사용 x, 공백 문자 사용 x

10. 변수 
	1)변수의 정의
		하나의 값을 저장할 수 있는 저장 공간
		값을 저장할 수 있는 메모리의 특성 번지에 붙여진 이름
		
	2) 자료형 어떤 자료를 담을 그릇
		메모리에 저장할 수 있는 값의 형태를 지정
		
		타입 변수이름 = 값;
		타입 변수이름;
		
		- 기본 자료형 : byte, short, long, float, double, boolean, char
		- 참조자료형 : 기본 자료형을 제외한 자료형
			       String, 배열, 클래스
